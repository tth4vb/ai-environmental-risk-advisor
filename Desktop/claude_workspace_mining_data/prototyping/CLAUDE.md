# CLAUDE.md - Prototyping Directory

This directory contains code prototypes for solution ideas being developed for the WRI Responsible Mining Data Platform project.

## Purpose

This is a place for building prototypes (in code) for solution ideas that we have for this project. Each solution gets its own dedicated folder with implementation code, documentation, and its own CLAUDE.md for specific guidance.

## Current Prototypes

### 1. AI Environmental Risk Advisor (`ai-environmental-risk-advisor/`)
- **Problem**: Local communities need independent, trustworthy sources of information on environmental risks for planned mining projects
- **Solution**: AI-powered platform providing automated environmental risk advice, reports, and customized community guides
- **Target Users**: Local community groups and CSOs near potential mining projects

### 2. Real-time Monitoring Platform (`realtime-monitoring-platform/`)
- **Problem**: Operating mining projects may have undisclosed environmental risks that communities lack tools to identify
- **Solution**: Map-based platform showing operating mining projects with real-time ESG and environmental impact data
- **Target Users**: Local community groups and CSOs near operating mining sites

## Development Guidelines

1. Each prototype should have its own folder with:
   - A CLAUDE.md file with specific implementation guidance
   - Clear README with setup instructions
   - Minimal viable implementation focusing on core value proposition
   - Example data or mock data for testing

2. Follow the main project principles from the parent CLAUDE.md:
   - Evidence-first approach
   - Rights & equity by design
   - Standards-aware implementation
   - Guard against misuse
   - Respect data sovereignty

3. Prototypes should be:
   - Quick to build and test
   - Focused on validating core assumptions
   - Designed for user testing with target personas
   - Built with production considerations in mind

## Adding New Prototypes

When adding a new solution prototype:
1. Create a new folder with a descriptive name
2. Add a CLAUDE.md specific to that prototype
3. Update this file with the new prototype information
4. Start with the simplest possible implementation that demonstrates value
